Expandable Objects

Switchboard makes use of an Expandable attribute in the composition root. This allows you to assign and modify the dependency injector properties in place, without having to switch windows or select a different object. This attribute can be used on any object field serialized in the editor to display that object with a dropdown menu, which displays all of the properties of the object in place. It cannot render custom property drawers, so the properties may not be rendered in the ideal fashion. Aside from that drawback, it allows you to inspect an object in place without have to explicitly select that object. Expandable objects fields can be nested, so there is no limit to the depth of expanding object references, except if a property has already been displayed in that branch of the inspector. Once a property has been displayed in a particular branch of nested expansions, it is no longer rendered. This avoids the potential for infinite recursion.
